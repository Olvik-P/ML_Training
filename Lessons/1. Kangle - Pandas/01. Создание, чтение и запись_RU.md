## Создание данных

В pandas есть два основных объекта: **DataFrame** и **Series**.

### DataFrame

**DataFrame** — это таблица. Она содержит массив отдельных записей, каждая из которых имеет определённое значение. Каждая запись соответствует строке (или записи) и столбцу.

Например, рассмотрим следующий простой DataFrame:

```python
pd.DataFrame({'Yes': [50, 21], 'No': [131, 2]})
```

|   | Yes | No  |
|---|-----|-----|
| 0 | 50  | 131 |
| 1 | 21  | 2   |

В этом примере запись «0, No» имеет значение 131. Запись «0, Yes» имеет значение 50 и так далее.

Записи DataFrame не ограничиваются целыми числами. Например, вот DataFrame, значения которого являются строками:

```python
pd.DataFrame({'Bob': ['I liked it.', 'It was awful.'], 'Sue': ['Pretty good.', 'Bland.']})
```

|   | Bob           | Sue            |
|---|---------------|----------------|
| 0 | I liked it.   | Pretty good.   |
| 1 | It was awful. | Bland.         |

Мы используем конструктор `pd.DataFrame()` для создания этих объектов DataFrame. Синтаксис объявления нового DataFrame — это словарь, ключами которого являются имена столбцов (в данном примере Bob и Sue), а значениями — списки записей. Это стандартный способ создания нового DataFrame, и именно с ним вы, скорее всего, столкнётесь.

Конструктор «словарь-список» присваивает значения меткам столбцов, но для меток строк использует просто возрастающий счёт от 0 (0, 1, 2, 3...). Иногда это приемлемо, но часто мы захотим назначать эти метки самостоятельно.

Список меток строк, используемых в DataFrame, называется **Index** (индекс). Мы можем присвоить ему значения с помощью параметра `index` в нашем конструкторе:

```python
pd.DataFrame({'Bob': ['I liked it.', 'It was awful.'], 
              'Sue': ['Pretty good.', 'Bland.']},
             index=['Product A', 'Product B'])
```

|           | Bob           | Sue            |
|-----------|---------------|----------------|
| Product A | I liked it.   | Pretty good.   |
| Product B | It was awful. | Bland.         |

### Series

**Series**, в отличие от DataFrame, представляет собой последовательность значений данных. Если DataFrame — это таблица, то Series — это список. И на самом деле вы можете создать её просто из списка:

```python
pd.Series([1, 2, 3, 4, 5])
```

```
0    1
1    2
2    3
3    4
4    5
dtype: int64
```

Series, по сути, является одним столбцом DataFrame. Таким образом, вы можете назначать метки строк для Series так же, как и раньше, используя параметр `index`. Однако у Series нет имени столбца, у неё есть только одно общее имя:

```python
pd.Series([30, 35, 40], index=['2015 Sales', '2016 Sales', '2017 Sales'], name='Product A')
```

```
2015 Sales    30
2016 Sales    35
2017 Sales    40
Name: Product A, dtype: int64
```

Series и DataFrame тесно связаны. Полезно думать о DataFrame как о наборе Series, «склеенных» вместе. Мы увидим больше об этом в следующем разделе данного руководства.

## Чтение файлов данных

Умение создавать DataFrame или Series вручную — удобно. Но по большей части мы не будем создавать свои данные вручную. Вместо этого мы будем работать с уже существующими данными.

Данные могут храниться в самых разных формах и форматах. Самый базовый из них — это простой CSV-файл. Когда вы открываете CSV-файл, вы получаете нечто подобное:

```
Product A,Product B,Product C,
30,21,9,
35,34,1,
41,11,11
```

Итак, CSV-файл — это таблица значений, разделённых запятыми. Отсюда и название: «Comma-Separated Values» (значения, разделённые запятыми), или CSV.

Давайте теперь отложим наши игрушечные наборы данных и посмотрим, как выглядит реальный набор данных, когда мы считываем его в DataFrame. Мы будем использовать функцию `pd.read_csv()` для чтения данных в DataFrame. Это делается так:

```python
wine_reviews = pd.read_csv("../input/wine-reviews/winemag-data-130k-v2.csv")
```

Мы можем использовать атрибут `shape`, чтобы проверить, насколько велик получившийся DataFrame:

```python
wine_reviews.shape
```

```
(129971, 14)
```

Итак, наш новый DataFrame содержит 130 000 записей, распределённых по 14 различным столбцам. Это почти 2 миллиона записей!
