# Типы данных и пропущенные значения

## Введение

В этом руководстве вы узнаете, как исследовать типы данных в DataFrame или Series. Вы также научитесь находить и заменять пропущенные значения.

Чтобы приступить к упражнению по этой теме, [нажмите здесь](https://www.kaggle.com/kernels/fork/1275161).

## Dtypes

Тип данных для столбца в DataFrame или Series называется **dtype**.

Вы можете использовать свойство [`dtype`](https://pandas.pydata.org/docs/reference/api/pandas.Series.dtype.html), чтобы получить тип конкретного столбца. Например, мы можем получить dtype столбца `price` в DataFrame `reviews`:

```python
reviews.price.dtype
```

```
dtype('float64')
```

Кроме того, свойство [`dtypes`](https://pandas.pydata.org/docs/reference/api/pandas.DataFrame.dtypes.html) возвращает dtype каждого столбца в DataFrame:

```python
reviews.dtypes
```

```
country        object
description    object
                ...  
variety        object
winery         object
Length: 13, dtype: object
```

Типы данных сообщают нам, как pandas хранит данные внутри. `float64` означает, что используется 64-битное число с плавающей точкой; `int64` — целое число того же размера, и так далее.

Одна особенность, которую стоит иметь в виду (и которая здесь хорошо видна), заключается в том, что столбцы, полностью состоящие из строк, не имеют собственного типа — им присваивается тип `object`.

Тип столбца можно преобразовать в другой, если такое преобразование имеет смысл, с помощью функции [`astype()`](https://pandas.pydata.org/docs/reference/api/pandas.Series.astype.html). Например, мы можем преобразовать столбец `points` из существующего типа `int64` в `float64`:

```python
reviews.points.astype('float64')
```

```
0         87.0
1         87.0
          ... 
129969    90.0
129970    90.0
Name: points, Length: 129971, dtype: float64
```

Индекс DataFrame или Series также имеет свой dtype:

```python
reviews.index.dtype
```

```
dtype('int64')
```

Pandas также поддерживает более экзотические типы данных, такие как категориальные данные и данные временных рядов. Поскольку эти типы используются реже, мы опустим их до более позднего раздела этого руководства.

## Пропущенные данные

Записям с пропущенными значениями присваивается значение `NaN` (сокращение от *«Not a Number»* — «не число»). По техническим причинам эти значения `NaN` всегда имеют тип `float64`.

Pandas предоставляет несколько методов для работы с пропущенными данными. Для выбора записей с `NaN` можно использовать [`pd.isnull()`](https://pandas.pydata.org/docs/reference/api/pandas.isnull.html) (или его дополнение [`pd.notnull()`](https://pandas.pydata.org/docs/reference/api/pandas.notnull.html)). Используется это следующим образом:

```python
reviews[pd.isnull(reviews.country)]
```

Замена пропущенных значений — распространённая операция. Pandas предоставляет очень удобный метод для этой задачи: [`fillna()`](https://pandas.pydata.org/docs/reference/api/pandas.DataFrame.fillna.html). `fillna()` предлагает несколько различных стратегий для обработки таких данных. Например, мы можем просто заменить каждое `NaN` на `"Unknown"`:

```python
reviews.region_2.fillna("Unknown")
```

```
0         Unknown
1         Unknown
           ...   
129969    Unknown
129970    Unknown
Name: region_2, Length: 129971, dtype: object
```

Или мы можем заполнить каждое пропущенное значение первым непустым значением, которое встречается после данной записи в базе данных. Это называется стратегией **обратного заполнения (backfill)**.

Кроме того, у нас может быть непустое значение, которое мы хотим заменить. Например, предположим, что с момента публикации этого набора данных рецензент Керин О'Киф изменила свой Twitter-handle с `@kerinokeefe` на `@kerino`. Один из способов отразить это в наборе данных — использовать метод [`replace()`](https://pandas.pydata.org/docs/reference/api/pandas.Series.replace.html):

```python
reviews.taster_twitter_handle.replace("@kerinokeefe", "@kerino")
```

```
0            @kerino
1         @vossroger
             ...    
129969    @vossroger
129970    @vossroger
Name: taster_twitter_handle, Length: 129971, dtype: object
```

Метод `replace()` стоит упомянуть здесь, потому что он удобен для замены пропущенных данных, которым в наборе данных присвоены некие сигнальные значения: такие как `"Unknown"`, `"Undisclosed"`, `"Invalid"` и так далее.
