## Введение

Выборка конкретных значений из DataFrame или Series pandas — это неотъемлемый шаг практически в любой операции с данными, которую ты будешь выполнять. Поэтому одна из первых вещей, которую нужно освоить при работе с данными в Python, — это умение быстро и эффективно выбирать нужные тебе точки данных.

Разве Series pandas не похожа на расширенный словарь? По сути, так оно и есть, поэтому неудивительно, что для получения одного конкретного значения нам нужно всего лишь ещё раз использовать оператор индексации `[]`:

```python
reviews['country'][0]
```

## Индексация в pandas

Оператор индексации и выборка через атрибуты удобны тем, что работают так же, как и в остальной экосистеме Python. Новичку их легко освоить и использовать. Однако у pandas есть собственные операторы доступа — `loc` и `iloc`. Для более продвинутых операций рекомендуется использовать именно их.

### Индексация на основе позиции

Индексация pandas работает в одной из двух парадигм. Первая — это индексация на основе позиции: выбор данных на основе их числовой позиции в данных. Этой парадигмы придерживается `iloc`.

Чтобы выбрать первую строку данных в DataFrame, мы можем использовать следующее:

```python
reviews.iloc[0]
```

```
country                                                    Italy
description    Aromas include tropical fruit, broom, brimston...
                                      ...                        
variety                                              White Blend
winery                                                   Nicosia
Name: 0, Length: 13, dtype: object
```

И `loc`, и `iloc` работают по принципу «сначала строка, потом столбец». Это противоположно тому, что мы делаем в обычном Python, где сначала идёт столбец, а потом строка.

Это означает, что выбирать строки немного проще, а столбцы — немного сложнее. Чтобы получить столбец с помощью `iloc`, мы можем сделать следующее:

```python
reviews.iloc[:, 0]
```

```
0            Italy
1         Portugal
             ...   
129969      France
129970      France
Name: country, Length: 129971, dtype: object
```

Сам по себе оператор `:`, который также пришёл из обычного Python, означает «всё». Однако в сочетании с другими селекторами его можно использовать для указания диапазона значений. Например, чтобы выбрать столбец `country` только из первой, второй и третьей строк, мы бы сделали так:

```python
reviews.iloc[:3, 0]
```

```
0       Italy
1    Portugal
2          US
Name: country, dtype: object
```

Или, чтобы выбрать только вторую и третью записи, мы бы сделали так:

```python
reviews.iloc[1:3, 0]
```

```
1    Portugal
2          US
Name: country, dtype: object
```

Также можно передать список:

```python
reviews.iloc[[0, 1, 2], 0]
```

```
0       Italy
1    Portugal
2          US
Name: country, dtype: object
```

Наконец, стоит знать, что в выборке можно использовать отрицательные числа. В этом случае отсчёт будет идти от конца значений. Например, вот последние пять элементов набора данных:

```python
reviews.iloc[-5:]
```

### Выборка на основе меток

Вторая парадигма выборки атрибутов — та, которой следует оператор `loc`: выборка на основе меток. В этой парадигме значение имеет не позиция данных, а значение индекса.

Например, чтобы получить первую запись в `reviews`, мы бы сделали следующее:

```python
reviews.loc[0, 'country']
```

```
'Italy'
```

`iloc` концептуально проще, чем `loc`, потому что он игнорирует индексы набора данных. Когда мы используем `iloc`, мы рассматриваем набор данных как большую матрицу (список списков), к которой нужно обращаться по позиции. `loc`, напротив, использует информацию в индексах для своей работы. Поскольку твой набор данных обычно имеет осмысленные индексы, обычно проще использовать `loc`. Вот пример операции, которая гораздо проще с `loc`:

```python
reviews.loc[:, ['taster_name', 'taster_twitter_handle', 'points']]
```

### Выбор между loc и iloc

При выборе или переходе между `loc` и `iloc` стоит помнить об одной «ловушке»: эти два метода используют немного разные схемы индексации.

`iloc` использует схему индексации из стандартной библиотеки Python, где первый элемент диапазона включается, а последний исключается. Таким образом, `0:10` выберет записи `0,...,9`. `loc`, в свою очередь, использует инклюзивную индексацию. То есть `0:10` выберет записи `0,...,10`.

Почему так? Помни, что `loc` может индексировать любые типы из стандартной библиотеки: например, строки. Если у нас есть DataFrame с индексами `Apples, ..., Potatoes, ...`, и мы хотим выбрать «все фрукты по алфавиту между Apples и Potatoes», то гораздо удобнее использовать `df.loc['Apples':'Potatoes']`, чем, скажем, `df.loc['Apples', 'Potatoet']` (буква `t` идёт после `s` в алфавите).

Это особенно сбивает с толку, когда индекс DataFrame представляет собой простой числовой список, например `0,...,1000`. В этом случае `df.iloc[0:1000]` вернёт 1000 записей, а `df.loc[0:1000]` вернёт 1001! Чтобы получить 1000 элементов с помощью `loc`, нужно взять диапазон на единицу меньше и запросить `df.loc[0:999]`.

В остальном семантика использования `loc` такая же, как и у `iloc`.

Например, предположим, что нас интересуют вина из Италии, качество которых выше среднего.

## Присваивание данных

```python
reviews['critic'] = 'everyone'
reviews['critic']

```
