## Сводные функции

Pandas предоставляет множество простых «сводных функций» (неофициальное название), которые преобразуют данные в полезный вид. Например, рассмотрим метод [`describe()`](https://pandas.pydata.org/docs/reference/api/pandas.DataFrame.describe.html):

```python
reviews.points.describe()
```

```
count    129971.000000
mean         88.447138
              ...      
75%          91.000000
max         100.000000
Name: points, Length: 8, dtype: float64
```

Этот метод формирует высокоуровневую сводку атрибутов указанного столбца. Он учитывает тип данных, то есть его вывод меняется в зависимости от типа входных данных. Приведённый выше вывод имеет смысл только для числовых данных; для строковых данных мы получим следующее:

```python
reviews.taster_name.describe()
```

```
count         103727
unique            19
top       Roger Voss
freq           25514
Name: taster_name, dtype: object
```

Если вам нужно получить какую-то конкретную простую сводную статистику по столбцу DataFrame или Series, обычно для этого находится подходящая функция pandas.

Например, чтобы узнать среднее значение начисленных баллов (то есть насколько хорошо оценивается вино в среднем), мы можем использовать функцию [`mean()`](https://pandas.pydata.org/docs/reference/api/pandas.Series.mean.html):

```python
reviews.points.mean()
```

```
88.44713820775404
```

Чтобы увидеть список уникальных значений, мы можем использовать функцию [`unique()`](https://pandas.pydata.org/docs/reference/api/pandas.Series.unique.html):

```python
reviews.taster_name.unique()
```

```
array(['Kerin O\'Keefe', 'Roger Voss', 'Paul Gregutt',
```

Чтобы увидеть список уникальных значений и то, как часто они встречаются в наборе данных, мы можем использовать метод [`value_counts()`](https://pandas.pydata.org/docs/reference/api/pandas.Series.value_counts.html):

## Отображения (Maps)

**Отображение (map)** — это термин, заимствованный из математики, обозначающий функцию, которая берёт одно множество значений и «отображает» их в другое множество значений. В науке о данных мы часто нуждаемся в создании новых представлений на основе существующих данных или в преобразовании данных из текущего формата в тот, который нам понадобится в дальнейшем. Отображения как раз и выполняют эту работу, что делает их чрезвычайно важными для решения ваших задач!

Существует два метода отображения, которые вы будете часто использовать.

[`map()`](https://pandas.pydata.org/docs/reference/api/pandas.Series.map.html) — первый и немного более простой. Например, предположим, что мы хотим перецентрировать оценки вин относительно нуля. Мы можем сделать это следующим образом:

```python
review_points_mean = reviews.points.mean()
reviews.points.map(lambda p: p - review_points_mean)
```

```
0        -1.447138
1        -1.447138
            ...   
129969    1.552862
129970    1.552862
Name: points, Length: 129971, dtype: float64
```

Функция, которую вы передаёте в [`map()`](https://pandas.pydata.org/docs/reference/api/pandas.Series.map.html), должна принимать одно значение из Series (в примере выше — значение баллов) и возвращать преобразованную версию этого значения. [`map()`](https://pandas.pydata.org/docs/reference/api/pandas.Series.map.html) возвращает новую Series, где все значения были преобразованы вашей функцией.

[`apply()`](https://pandas.pydata.org/docs/reference/api/pandas.DataFrame.apply.html) — это эквивалентный метод, если мы хотим преобразовать целый DataFrame, вызывая пользовательский метод для каждой строки.

```python
def remean_points(row):
    row.points = row.points - review_points_mean
    return row

reviews.apply(remean_points, axis='columns')
```

Если бы мы вызвали `reviews.apply()` с параметром `axis='index'`, то вместо передачи функции для преобразования каждой строки нам нужно было бы передать функцию для преобразования каждого столбца.

Обратите внимание, что [`map()`](https://pandas.pydata.org/docs/reference/api/pandas.Series.map.html) и [`apply()`](https://pandas.pydata.org/docs/reference/api/pandas.DataFrame.apply.html) возвращают новые преобразованные Series и DataFrame соответственно. Они не изменяют исходные данные, для которых были вызваны. Если мы посмотрим на первую строку `reviews`, то увидим, что она всё ещё содержит исходное значение баллов.

```python
reviews.head(1)
```

| country | description | designation | points | price | province | region_1 | region_2 | taster_name | taster_twitter_handle | title | variety | winery |
|---|---|---|---|---|---|---|---|---|---|---|---|---|
| 0 | Italy | Aromas include tropical fruit, broom, brimston... | Vulkà Bianco | 87 | NaN | Sicily & Sardinia | Etna | NaN | Kerin O'Keefe | @kerinokeefe | Nicosia 2013 Vulkà Bianco (Etna) | White Blend | Nicosia |

Pandas предоставляет множество распространённых операций отображения в виде встроенных методов. Например, вот более быстрый способ перецентрировать наш столбец с баллами:

```python
review_points_mean = reviews.points.mean()
reviews.points - review_points_mean
```

```
0        -1.447138
1        -1.447138
            ...   
129969    1.552862
129970    1.552862
Name: points, Length: 129971, dtype: float64
```

В этом коде мы выполняем операцию между множеством значений слева (все элементы Series) и одним значением справа (среднее значение). Pandas анализирует это выражение и понимает, что мы хотим вычесть это среднее значение из каждого элемента набора данных.

Pandas также поймёт, что делать, если мы выполняем такие операции между Series одинаковой длины. Например, простой способ объединить информацию о стране и регионе в наборе данных — сделать следующее:

```python
reviews.country + " - " + reviews.region_1
```

```
0            Italy - Etna
1                     NaN
                ...       
129969    France - Alsace
129970    France - Alsace
Length: 129971, dtype: object
```

Эти операторы работают быстрее, чем [`map()`](https://pandas.pydata.org/docs/reference/api/pandas.Series.map.html) или [`apply()`](https://pandas.pydata.org/docs/reference/api/pandas.DataFrame.apply.html), поскольку используют встроенные в pandas механизмы ускорения. Все стандартные операторы Python (`>`, `<`, `==` и так далее) работают подобным образом.

Однако они не так гибки, как [`map()`](https://pandas.pydata.org/docs/reference/api/pandas.Series.map.html) или [`apply()`](https://pandas.pydata.org/docs/reference/api/pandas.DataFrame.apply.html), которые могут выполнять более сложные действия, например применять условную логику, что невозможно сделать с помощью одного лишь сложения и вычитания.
