# Группировка и сортировка

Повысьте уровень своих инсайтов. Чем сложнее набор данных, тем важнее это умение.

Введение
Функции-отображения (maps) позволяют преобразовывать данные в DataFrame или Series по одному значению за раз для целого столбца. Однако часто нам нужно сгруппировать данные, а затем выполнить какое-либо специфическое действие для каждой группы.

Как вы узнаете, для этого используется операция `groupby()`. Мы также рассмотрим некоторые дополнительные темы, такие как более сложные способы индексации DataFrame, а также сортировку данных.

Чтобы начать упражнение по этой теме, пожалуйста, нажмите здесь.

## Групповой анализ
Одна из функций, которую мы активно использовали до сих пор — это `value_counts()`. Мы можем воспроизвести то, что делает `value_counts()`, следующим образом:

```python
reviews.groupby('points').points.count()
```

```
points
80     397
81     692
       ... 
99      33
100     19
Name: points, Length: 21, dtype: int64
```

`groupby()` создаёт группу обзоров, которым были присвоены одинаковые значения баллов для указанных вин. Затем для каждой из этих групп мы взяли столбец `points` и подсчитали, сколько раз он встретился. `value_counts()` — это просто сокращённый вариант этой операции `groupby()`.

С этими данными мы можем использовать любые сводные функции, которые применяли ранее. Например, чтобы получить самое дешёвое вино в каждой категории баллов, мы можем сделать следующее:

```python
reviews.groupby('points').price.min()
```

```
points
80      5.0
81      5.0
        ... 
99     44.0
100    80.0
Name: price, Length: 21, dtype: float64
```

Каждую созданную группу можно рассматривать как срез нашего DataFrame, содержащий только данные с соответствующими значениями. Этот DataFrame доступен нам напрямую через метод `apply()`, и мы можем манипулировать данными любым удобным способом. Например, вот один из способов выбрать название первого вина, оценённого от каждой винодельни в наборе данных:

```python
reviews.groupby('winery').apply(lambda df: df.title.iloc[0])
```

```
                                  ...
```

Для ещё более тонкого контроля вы можете группировать данные по нескольким столбцам. Например, вот как мы можем выбрать лучшее вино по стране и провинции:

```python
reviews.groupby(['country', 'province']).apply(lambda df: df.loc[df.points.idxmax()])
```

Ещё один метод `groupby()`, заслуживающий внимания — это `agg()`, который позволяет запускать сразу несколько различных функций на вашем DataFrame одновременно. Например, мы можем сгенерировать простую статистическую сводку набора данных следующим образом:

```python
reviews.groupby(['country']).price.agg([len, min, max])
```

Эффективное использование `groupby()` позволит вам делать множество действительно мощных вещей с вашим набором данных.

## Мультииндексы
Во всех примерах, которые мы видели до сих пор, мы работали с объектами DataFrame или Series, имеющими индекс с одной меткой. `groupby()` отличается тем, что в зависимости от выполняемой операции иногда приводит к тому, что называется мультииндексом.

Мультииндекс отличается от обычного индекса тем, что имеет несколько уровней. Например:

```python
countries_reviewed = reviews.groupby(['country', 'province']).description.agg([len])
countries_reviewed
```

|       | len |
|-------|-----|
| **country** | **province** | |
| Argentina | Mendoza Province | 3264 |
| | Other | 536 |
| ... | ... | ... |
| Uruguay | San Jose | 3 |
| | Uruguay | 24 |

*425 rows × 1 columns*

```python
mi = countries_reviewed.index
type(mi)
```

```
pandas.core.indexes.multi.MultiIndex
```

Мультииндексы имеют несколько методов для работы со своей многоуровневой структурой, которые отсутствуют у одноуровневых индексов. Они также требуют двух уровней меток для извлечения значения. Работа с мультииндексным выводом — это распространённая «ловушка» для новых пользователей pandas.

Варианты использования мультииндекса подробно описаны вместе с инструкциями по их применению в разделе MultiIndex / Advanced Selection документации pandas.

Однако в целом метод мультииндекса, который вы будете использовать чаще всего — это метод преобразования обратно в обычный индекс, а именно `reset_index()`:

```python
countries_reviewed.reset_index()
```

|   | country | province | len |
|---|---------|----------|-----|
| 0 | Argentina | Mendoza Province | 3264 |
| 1 | Argentina | Other | 536 |
| ... | ... | ... | ... |
| 423 | Uruguay | San Jose | 3 |
| 424 | Uruguay | Uruguay | 24 |

*425 rows × 3 columns*

## Сортировка
Взглянув снова на `countries_reviewed`, мы видим, что группировка возвращает данные в порядке индекса, а не в порядке значений. То есть при выводе результата `groupby` порядок строк зависит от значений в индексе, а не в данных.

Чтобы получить данные в нужном нам порядке, мы можем отсортировать их самостоятельно. Для этого удобно использовать метод `sort_values()`.

```python
countries_reviewed = countries_reviewed.reset_index()
countries_reviewed.sort_values(by='len')
```

|   | country | province | len |
|---|---------|----------|-----|
| 179 | Greece | Muscat of Kefallonian | 1 |
| 192 | Greece | Sterea Ellada | 1 |
| ... | ... | ... | ... |
| 415 | US | Washington | 8639 |
| 392 | US | California | 36247 |

*425 rows × 3 columns*

`sort_values()` по умолчанию выполняет сортировку по возрастанию, где наименьшие значения идут первыми. Однако чаще всего нам нужна сортировка по убыванию, где большие числа идут первыми. Это делается так:

```python
countries_reviewed.sort_values(by='len', ascending=False)
```

|   | country | province | len |
|---|---------|----------|-----|
| 392 | US | California | 36247 |
| 415 | US | Washington | 8639 |
| ... | ... | ... | ... |
| 63 | Chile | Coelemu | 1 |
| 149 | Greece | Beotia | 1 |

*425 rows × 3 columns*

Для сортировки по значениям индекса используйте сопутствующий метод `sort_index()`. Этот метод имеет те же аргументы и порядок по умолчанию:

```python
countries_reviewed.sort_index()
```

|   | country | province | len |
|---|---------|----------|-----|
| 0 | Argentina | Mendoza Province | 3264 |
| 1 | Argentina | Other | 536 |
| ... | ... | ... | ... |
| 423 | Uruguay | San Jose | 3 |
| 424 | Uruguay | Uruguay | 24 |

*425 rows × 3 columns*

И наконец, имейте в виду, что вы можете сортировать сразу по нескольким столбцам:

```python
countries_reviewed.sort_values(by=['country', 'len'])
```

|   | country | province | len |
|---|---------|----------|-----|
| 1 | Argentina | Other | 536 |
| 0 | Argentina | Mendoza Province | 3264 |
| ... | ... | ... | ... |
| 424 | Uruguay | Uruguay | 24 |
| 419 | Uruguay | Canelones | 43 |

*425 rows × 3 columns*
