## Что такое валидация модели
Вам захочется оценивать почти каждую модель, которую вы когда-либо построите. В большинстве (хотя и не во всех) приложений релевантной мерой качества модели является **прогностическая точность**. Другими словами, будут ли прогнозы модели близки к тому, что происходит на самом деле.

Многие совершают огромную ошибку при измерении прогностической точности. Они делают прогнозы на своих обучающих данных и сравнивают эти прогнозы с целевыми значениями в обучающих данных. Вы увидите проблему этого подхода и как её решить чуть позже, но давайте сначала подумаем, как бы мы это сделали.

Сначала нужно обобщить качество модели в понятной форме. Если вы сравните прогнозируемые и фактические стоимости домов для 10 000 домов, вы, скорее всего, обнаружите смесь хороших и плохих прогнозов. Просматривать список из 10 000 прогнозируемых и фактических значений бессмысленно. Нам нужно свести это к единой метрике.

Существует множество метрик для обобщения качества модели, но мы начнём с одной под названием **Средняя абсолютная ошибка** (также называемая MAE). Давайте разберём эту метрику, начиная с последнего слова — ошибка.

Ошибка прогноза для каждого дома:

```
error = actual − predicted
```

Таким образом, если дом стоит 150 000 долларов, а вы предсказали, что он будет стоить 100 000 долларов, ошибка составляет 50 000 долларов.

В метрике MAE мы берём абсолютное значение каждой ошибки. Это преобразует каждую ошибку в положительное число. Затем мы берём среднее этих абсолютных ошибок. Это наша мера качества модели. Простыми словами это можно выразить так:

**В среднем наши прогнозы отклоняются примерно на X.**

Чтобы рассчитать MAE, сначала нужна модель. Она построена в скрытой ячейке ниже, которую вы можете просмотреть, нажав кнопку кода.

Как только у нас есть модель, вот как мы вычисляем среднюю абсолютную ошибку:

```python
from sklearn.metrics import mean_absolute_error

predicted_home_prices = melbourne_model.predict(X)
mean_absolute_error(y, predicted_home_prices)
```

## Проблема с оценками «внутри выборки»
Только что вычисленную меру можно назвать оценкой «внутри выборки». Мы использовали одну «выборку» домов как для построения модели, так и для её оценки. Вот почему это плохо.

Представьте, что на большом рынке недвижимости цвет двери не связан с ценой дома.

Однако в выборке данных, которую вы использовали для построения модели, все дома с зелёными дверями были очень дорогими. Задача модели — находить закономерности, предсказывающие цены на дома, поэтому она увидит эту закономерность и всегда будет предсказывать высокие цены для домов с зелёными дверями.

Поскольку эта закономерность была получена из обучающих данных, модель будет казаться точной на обучающих данных.

Но если эта закономерность не сохранится, когда модель увидит новые данные, модель окажется очень неточной на практике.

Поскольку практическая ценность моделей заключается в прогнозировании на новых данных, мы измеряем производительность на данных, которые не использовались для построения модели. Самый простой способ сделать это — исключить часть данных из процесса построения модели, а затем использовать их для проверки точности модели на данных, которые она раньше не видела. Эти данные называются **валидационными данными**.

## Реализация
Библиотека scikit-learn имеет функцию `train_test_split` для разделения данных на две части. Мы используем часть этих данных в качестве обучающих данных для подгонки модели, а другую часть — в качестве валидационных данных для вычисления средней абсолютной ошибки.

Вот код:

```python
from sklearn.model_selection import train_test_split

# разделить данные на обучающие и валидационные, как для признаков, так и для целевой переменной
# Разделение основано на генераторе случайных чисел. Передача числового значения
# аргументу random_state гарантирует, что мы получим одно и то же разделение каждый раз,
# когда запускаем этот скрипт.
train_X, val_X, train_y, val_y = train_test_split(X, y, random_state = 0)
# Определить модель
melbourne_model = DecisionTreeRegressor()
# Обучить модель
melbourne_model.fit(train_X, train_y)

# получить прогнозируемые цены на валидационных данных
val_predictions = melbourne_model.predict(val_X)
print(mean_absolute_error(val_y, val_predictions))
```

Ваша средняя абсолютная ошибка для данных внутри выборки составляла около 500 долларов. Для данных вне выборки она превышает 250 000 долларов.

Это разница между моделью, которая почти идеально точна, и моделью, непригодной для большинства практических целей. Для справки: средняя стоимость дома в валидационных данных составляет 1,1 миллиона долларов. Таким образом, ошибка на новых данных составляет около четверти средней стоимости дома.

Есть много способов улучшить эту модель, например, экспериментировать с поиском лучших признаков или различных типов моделей.
