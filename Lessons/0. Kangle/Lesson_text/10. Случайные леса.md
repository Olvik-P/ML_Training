# Случайные леса
Использование более сложного алгоритма машинного обучения.

## Введение

Деревья решений ставят перед вами сложный выбор. Глубокое дерево с большим количеством листьев будет переобучаться, потому что каждое предсказание основывается на исторических данных всего лишь нескольких домов, попавших в соответствующий лист. Но неглубокое дерево с малым числом листьев будет плохо работать, поскольку не сможет уловить многие различия в исходных данных.

Даже самые современные методы моделирования сегодня сталкиваются с этим противоречием между недообучением и переобучением. Однако многие модели содержат clever идеи, которые могут привести к лучшей производительности. В качестве примера мы рассмотрим случайный лес.

Случайный лес использует множество деревьев и делает предсказание, усредняя предсказания каждого из составляющих деревьев. Как правило, он обладает значительно более высокой прогнозной точностью, чем одно дерево решений, и хорошо работает с параметрами по умолчанию. Если вы продолжите изучать моделирование, вы сможете познакомиться с другими моделями, имеющими ещё лучшую производительность, но многие из них чувствительны к правильному подбору параметров.

## Пример

Вы уже несколько раз видели код загрузки данных. После загрузки данных у нас есть следующие переменные:

- `train_X`
- `val_X`
- `train_y`
- `val_y`

Мы строим модель случайного леса аналогично тому, как строили дерево решений в scikit-learn — на этот раз используя класс `RandomForestRegressor` вместо `DecisionTreeRegressor`.

```python
from sklearn.ensemble import RandomForestRegressor
from sklearn.metrics import mean_absolute_error

forest_model = RandomForestRegressor(random_state=1)
forest_model.fit(train_X, train_y)
melb_preds = forest_model.predict(val_X)
print(mean_absolute_error(val_y, melb_preds))
```

## Заключение

Вероятно, есть возможность для дальнейшего улучшения, но это уже значительный прогресс по сравнению с наилучшей ошибкой дерева решений в 250 000. Существуют параметры, позволяющие изменять производительность случайного леса, подобно тому как мы меняли максимальную глубину у одиночного дерева. Однако одно из лучших свойств моделей случайного леса заключается в том, что они, как правило, работают достаточно хорошо даже без такой настройки.
