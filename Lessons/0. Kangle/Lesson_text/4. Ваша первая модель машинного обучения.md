## Выбор данных для моделирования
Ваш набор данных содержит слишком много переменных, чтобы их можно было легко осмыслить или даже красиво вывести. Как можно сократить это ошеломляющее количество данных до чего-то понятного?

Мы начнем с выбора нескольких переменных, используя нашу интуицию. В последующих курсах будут показаны статистические методы для автоматического определения приоритетов переменных.

Чтобы выбрать переменные/столбцы, нам нужно увидеть список всех столбцов в наборе данных. Это делается с помощью свойства `columns` DataFrame (нижняя строка кода ниже).

- Точечная нотация, которую мы используем для выбора «целевой переменной»
- Выбор с помощью списка столбцов, который мы используем для выбора «признаков»

## Выбор целевой переменной
Вы можете извлечь переменную с помощью точечной нотации. Этот отдельный столбец хранится в Series, что в целом похоже на DataFrame только с одним столбцом данных.

Мы будем использовать точечную нотацию для выбора столбца, который хотим предсказать, называемого целевой переменной. По соглашению целевая переменная обозначается как `y`. Таким образом, код для сохранения цен на дома в данных по Мельбурну будет:

```python
y = melbourne_data.Price
```

## Выбор «признаков»
Столбцы, которые подаются на вход нашей модели (и позже используются для прогнозирования), называются «признаками». В нашем случае это будут столбцы, используемые для определения цены дома. Иногда вы будете использовать все столбцы, кроме целевого, в качестве признаков. В других случаях лучше использовать меньше признаков.

Пока мы построим модель только с несколькими признаками. Позже вы увидите, как итерировать и сравнивать модели, построенные с разными признаками.

Мы выбираем несколько признаков, предоставляя список имен столбцов в квадратных скобках. Каждый элемент в этом списке должен быть строкой (в кавычках).

Вот пример:

```python
melbourne_features = ['Rooms', 'Bathroom', 'Landsize', 'Lattitude', 'Longtitude']
```

По соглашению эти данные называются `X`.

```python
X = melbourne_data[melbourne_features]
```

## Построение вашей модели
Вы будете использовать библиотеку scikit-learn для создания ваших моделей. При написании кода эта библиотека записывается как `sklearn`, как вы увидите в примере кода. Scikit-learn, безусловно, самая популярная библиотека для моделирования типов данных, обычно хранящихся в DataFrames.

Шаги по созданию и использованию модели:

1. **Определение**: Какого типа будет модель? Дерево решений? Другой тип модели? Также указываются некоторые другие параметры типа модели.
2. **Обучение**: Захват закономерностей из предоставленных данных. Это суть моделирования.
3. **Прогнозирование**: То, что звучит как
4. **Оценка**: Определение точности прогнозов модели.

Вот пример определения модели дерева решений с помощью scikit-learn и её обучения на признаках и целевой переменной.

```python
from sklearn.tree import DecisionTreeRegressor

# Определяем модель. Указываем число для random_state, чтобы гарантировать одинаковые результаты при каждом запуске
melbourne_model = DecisionTreeRegressor(random_state=1)

# Обучаем модель
melbourne_model.fit(X, y)
```

```
DecisionTreeRegressor(random_state=1)
```

Многие модели машинного обучения допускают некоторую случайность при обучении модели. Указание числа для `random_state` гарантирует, что вы получите одинаковые результаты при каждом запуске. Это считается хорошей практикой. Вы можете использовать любое число, и качество модели не будет существенно зависеть от того, какое именно значение вы выберете.

Теперь у нас есть обученная модель, которую мы можем использовать для прогнозирования.

На практике вы захотите делать прогнозы для новых домов, появляющихся на рынке, а не для домов, цены на которые у нас уже есть. Но мы сделаем прогнозы для первых нескольких строк обучающих данных, чтобы увидеть, как работает функция `predict`.

